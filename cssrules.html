CSS Rules

CSS allows programmers to customize styles for different devices and screen sizes. This improves website responsiveness and user-friendliness. As an example, the same webpage may appear one way on a mobile device and another way on a desktop PC.

@Rules:

Definition:
At-rules are essentially CSS declarations that teach the CSS how to act under specific situations. i.e., At-rules enable developers to modify the style or design of webpages based on certain situations. Every At-rule command starts with '@', followed by an identifier, and ends with a semicolon ';' or the following CSS block.
Syntax:

  • Regular At-rules:
    @identifier (condition);
  • Nested At-rules:
    @identifier (condition) {}
    • At-Rules Description
    @charset Specifies the set of characters to be used on the webpage. Commonly UTF-8 is used.
    @import Imports styles in CSS or rules from other CSS files on the local system or from the internet.
    @namespace Tells the CSS engine that all the XML namespaces are to be used in the CSS style sheet.
    @media Applies CSS rules based on conditions like screen size. Used for responsive design.
    @support Determines browser support capabilities. If supported, the style is applied (e.g., grids).
    @page Modifies CSS properties for printing pages.
    @font-face Defines a custom font to display text. You provide the font name and its source URL.
    @keyframes Defines animations by gradually changing CSS styles from one state to another.
    @counter-style Lets you design custom counter styles for lists or numbering.
